🔧 Field Tips

Mark with a fine pen — thick lines = ±0.5mm error per mark

Align the BACK OF BEND arrow with your mark, not the front

For all offset bends — rotate exactly 180° between marks. Use a spirit level or laser

Rolling offsets — set roll angle with a digital protractor before every bend

Allow 3–5mm extra on first cuts — you can trim, you cannot add

Stainless work-hardens — one smooth motion per bend, never rock back and forth

Fault Finding
Diagnostic steps for common E&I faults
📡 Transmitter / Loop Faults
Transmitter stuck at 0% / 4mA▼
1
Check supply voltage at terminals - should be 12 to 36VDC. Low voltage = loop power issue.
2
Measure mA in series. 0mA or less than 3.6mA = open circuit or hardware fault.
3
Check cable continuity and terminal connections for open circuit.
4
Check process - is actual PV at or below LRV? Instrument may be reading correctly.
5
DP transmitters: check manifold valves and impulse lines for blockages.
Transmitter pegged high / 20mA+▼
1
Verify process value independently - is vessel actually full?
2
DP: check for blocked HP impulse line - blocked high side reads high.
3
Review URV - may be set too low for current process range.
4
Check for short on sensing element - can drive output to saturation.
Noisy or erratic signal▼
1
Shield grounded at ONE end only - dual grounding causes ground loops and noise.
2
Cable routed near VFDs, contactors or HV cables - reroute or use SWA cable.
3
Inspect all terminals for loose connections or corrosion.
4
Increase transmitter damping - 1 to 5 seconds is a typical starting point.
HART communication failure▼
1
Loop resistance must be 230 to 1100 ohm. Add resistor if DCS input is too low.
2
Confirm supply voltage is at least 12VDC at device under load.
3
Long cable runs over 1.5km attenuate HART signal - use a HART repeater.
4
Verify HART revision - some DCS cards only support HART 5 not HART 7.
Temperature TX reading wrong▼
1
Verify TC type matches TX config - J, K, T, E type mismatch causes large offset errors.
2
Check CJC - cold junction compensation sensor fault offsets all readings.
3
Extension cable must match TC type - copper cable introduces junction errors.
4
RTD: check 2/3/4-wire config matches. 2-wire adds lead resistance error.
🔌 Control Valve / Actuator
Valve not responding to signal▼
1
Confirm 4 to 20mA signal reaching positioner - measure at positioner terminals.
2
Check instrument air - clean, dry, regulated at 1.4 to 5.5 bar.
3
Stroke manually if safe to do so - determines if mechanical or signal fault.
4
Check I/P output with gauge - should track 0.2 to 1.0 bar for 4 to 20mA input.
Valve hunting or oscillating▼
1
Check PID tuning - reduce gain or increase integral time. Hunting usually means over-tuned loop.
2
Reduce positioner proportional gain via configuration.
3
Check for air supply pressure fluctuations.
4
Inspect for mechanical backlash in linkage or stem - excessive play causes oscillation.

🔬 SmartLoop Testing
SmartLoop tests both fibres in one OTDR shot — requires a loop-back at the far end. Use two launch cords and one tail cord.

Setup: OTDR → Launch cord A → Link fibre 1 → Loopback → Link fibre 2 → Tail cord B → OTDR

Benefit: Tests both directions simultaneously, reduces test time by 50%

Key setting: Enable SmartLoop in test setup. Set loop-back reflectance threshold to detect the mid-point splice/connector

⚖️ Launch & Tail Compensation
Compensation removes the launch/tail cords from the measurement so only the link under test is reported.
How to set launch compensation▼
1
Connect launch cord to OTDR, leave far end open or connect to known reference.
2
Run a test. In the event table, identify the far end of the launch cord (first connector event after the dead zone).
3
Set the launch compensation marker at this event. The OTDR will zero distance here.
How to set tail compensation▼
1
Connect tail cord to far end of link.
2
Identify the connector event where the tail cord begins in the event table.
3
Set the tail compensation marker here. Loss at this connector is excluded from the result.
🔄 Bi-Directional Testing
Required for Tier 2 certification (TIA-568, ISO 14763-3). Averages the OTDR readings from both ends to cancel out gain/loss anomalies caused by splice direction.
StepAction
1Test A→B with launch + tail compensation set. Record all event losses.
2Swap ends — test B→A. Record all event losses.
3Average each event: (A→B loss + B→A loss) ÷ 2
4Compare averaged loss to pass/fail limit for the fibre grade.
5OptiFiber Pro SmartLoop does this automatically in one shot.

📡 OTDR Event Identification Guide
Each event type has a characteristic shape on the OTDR trace. Use this guide to identify what you are looking at.
💥 Fibre Break

Characteristics: Sudden vertical drop to noise floor. No further signal. Possible reflection spike if air gap present.

Causes: Crushed cable, rodent damage, over-bend, anchor clamp failure, dig-in.

Action: Locate break using OTDR distance reading. Expose, cut back, re-splice or replace section.

🔴 Dirty Connector

Characteristics: High reflective event with loss greater than 0.5dB at connector. May show as poor ORL.

Causes: Contamination on ferrule end face — oil, dust, moisture, APC/UPC mismatch.

Action: Inspect with fibre scope (≥200×). Clean with IEC 61300-3-35 method (dry then wet). Re-inspect before reconnecting.

📉 Poor Splice

Characteristics: Non-reflective step loss >0.3dB. No reflection. Loss varies with direction (bi-di average reveals true loss).

Causes: Bad cleave, dirty fibre, misaligned cores, worn electrodes, incorrect arc settings.

Action: Re-splice. Check cleave angle (<0.5°), clean fibre, run electrode conditioning, check arc calibration.

👻 Ghost Event

Characteristics: Apparent event at exactly 2× the distance of a strong reflective event. No loss. Looks like a connector but isn't.

Cause: Strong reflection bounces back to OTDR laser, re-reflects, and appears as a second event at double the distance.

Action: Confirm by checking the event is at exactly 2× a known reflective event. Add a short attenuator pad after the OTDR port to suppress. Not a real fault.

〰️ Macrobend

Characteristics: Gradual increase in attenuation slope (trace gets steeper). Worse at 1550nm than 1310nm. May show as non-reflective loss.

Causes: Cable bent below minimum bend radius (typically 20× OD), kinking in duct, tight radius at termination box.

Action: Locate by comparing 1310nm and 1550nm traces — the bend loss ratio identifies the location. Relieve bend.

📊 High Attenuation Section

Characteristics: Section of trace with steeper slope than the rest. Loss per km exceeds fibre specification.

Causes: Microbends from crush damage, water ingress, UV degradation, wrong fibre type in link.

Action: Compare measured dB/km to spec. Identify section boundaries from trace. Inspect cable physically if accessible.

🔀 Splitter Detection

Characteristics: Large non-reflective loss (3.5dB for 1:2, 7dB for 1:4, 10.5dB for 1:8, 14dB for 1:16). No reflection. Trace continues after loss step.

Use: In PON/FTTx networks splitters are expected. On an enterprise link, an unexpected splitter indicates wrong cable route or unauthorised tap.

⚡ AFL Fujikura — Arc Calibration
When to calibrate▼
1
Run arc calibration at start of each job, after electrode replacement, after significant altitude or temperature change.
2
Also calibrate if splice losses are consistently higher than expected or splicer is showing inconsistent results.
Arc calibration procedure▼
1
Prepare two clean, well-cleaved fibre ends (cleave angle <0.5°).
2
Load fibres and select ARC CAL from the menu.
3
Splicer fires test arcs and adjusts arc power automatically. Do not open lid during calibration.
4
Calibration complete when splicer confirms. Estimated loss should return to <0.02dB typical.

✂️ Fibre Preparation & Cleaver Maintenance

Strip: Remove coating to exact length required (typically 30–40mm). Use the correct size stripper for the coating type.

Clean: Wipe stripped bare glass with IPA on lint-free wipe. One direction only. Never reuse wipe on same fibre.

Cleave: Use precision cleaver. Cleave angle must be <0.5° for SM, <1.0° for MM. Inspect under scope before loading.

Cleaver blade: Rotate blade position every 1000 cleaves. Replace blade when cleave quality deteriorates.

Waste fibre: Collect all glass shards in a sealed container. Never leave on bench — bare glass is invisible and dangerous.

🔍 Splice Fault Finder — Fujikura Splicer
High Splice Loss (>0.10 dB)▼
1
Check cleave angle — must be <0.5°. Re-cleave if poor.
2
Clean bare fibre again — even microscopic contamination causes loss.
3
Run arc calibration — electrodes may be degraded.
4
Check fibre type match — splicing dissimilar fibres (e.g. OM3 to OM4) will inherently increase loss.
5
Re-splice. If consistently high, replace electrodes and recalibrate.
Thick Splice (bulge visible in splicer camera)▼
1
Caused by too much arc power or too long an arc duration — fibres pushed too far together.
2
Run arc calibration. Reduce arc power in manual mode if available.
3
Check fibre overlap setting (usually auto). Re-splice.
Thin Splice (waist/neck visible)▼
1
Caused by too little arc power or too short a duration — fibres not fully fused, drawn apart.
2
Run arc calibration. Check for draught or wind affecting arc (work indoors or in enclosure).
3
Thin splices are mechanically weak — always re-splice even if loss appears acceptable.
Bubble in Splice▼
1
Contamination trapped in the arc — fibre not cleaned properly, or moisture present.
2
Re-clean, re-cleave, and re-splice. Never reuse a cleaved fibre end — re-cleave fresh.
3
Bubble splices will fail mechanically under tension — always re-splice.
Bad Cleave / Lip on Fibre End▼
1
Rotate cleaver blade to a fresh position — current position is worn.
2
Check fibre is seated correctly in the cleaver groove — must be fully down.
3
Replace cleaver blade if rotating positions doesn't improve quality.
Worn / Oxidised Electrodes▼
1
Signs: arc calibration repeatedly fails, high loss even on clean splices, arc appears uneven in camera.
2
Run electrode conditioning cycle first — may restore performance.
3
If conditioning doesn't resolve: replace electrodes, condition, then calibrate before splicing.
📋 OptiFiber Pro — SmartLoop Procedure
1
Inspect and clean all connectors to be used (OTDR port, launch cords, tail cord) with approved cleaner and wipe.
2
Connect launch cord A to OTDR port. Connect other end to Fibre 1 of the link.
3
At the far end, connect Fibre 1 and Fibre 2 together with a loopback patch cord.
4
Connect Fibre 2 back to the OTDR via tail cord B.
5
In OptiFiber Pro, select SmartLoop mode. Enter fibre type, wavelengths, and expected link length.
6
Set launch compensation to end of launch cord A, tail compensation to start of tail cord B.
7
Run test. Review event table. Save result with cable ID and date.
📋 Fujikura Fusion Splice Procedure
1
Select correct splice program for fibre type. Run arc calibration if not done today.
2
Thread heat-shrink protector onto fibre before stripping.
3
Strip coating to required length using correct strippers. Remove all coating — check under UV light if needed.
4
Clean bare glass twice with fresh IPA wipe. One direction only.
5
Cleave fibre. Inspect cleave angle on splicer camera — reject anything above 0.5°.
6
Load both fibres into splicer without touching bare glass. Close lid.
7
Press splice. Review estimated loss on screen. Accept if ≤0.05dB SM, ≤0.10dB MM. Re-splice if above limit.
8
Remove spliced fibre carefully. Slide heat-shrink over splice. Centre in heater. Run heat cycle. Allow to cool.
9
Place in splice tray. Coil fibre with no tight bends. Secure with fixings provided in closure/tray.
📋 Acceptance Testing Workflow
1
Obtain link schedule — cable IDs, fibre count, connector types, required standard (TIA/ISO).
2
Calculate loss budget for each link before testing — sets your pass/fail threshold.
3
Inspect all connectors at both ends before connecting to test equipment.
4
Run insertion loss test (Tier 1) first — OLTS / light source and power meter.
5
Run OTDR (Tier 2) from both ends or SmartLoop. Record all events.
6
Compare results to budget. Document pass/fail per fibre per wavelength.
7
Remediate failures — re-clean, re-splice, re-terminate as required. Re-test after each fix.
8
Export and save all results from OptiFiber Pro. Provide to client with cable schedule.
